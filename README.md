# 崩壞：星穹鐵道 限定卡池抽卡模擬器

這是一個基於大數據機率模型與蒙地卡羅方法（Monte Carlo method）所開發的《崩壞：星穹鐵道》網頁版抽卡模擬器。透過高達數十萬次的快速模擬，幫助玩家精確推算出在當前資源與墊抽狀態下，達成目標配置（如 2+1、6+5）的真實機率分布。

## ⚙️ 核心模擬機制與規則

本模擬器嚴格遵循遊戲內的機率設定、隱藏軟保底機制，以及最真實的「未熄的星芒」資源回收系統，詳細規則如下：

### 一、 五星角色池（限定活動躍遷）
* **基礎機率**：單抽 0.6%。
* **硬保底**：90 抽必中五星。
* **軟保底（隱藏機制）**：從第 74 抽開始，每抽中獎機率增加 6%（第 74 抽 6.6%、第 75 抽 12.6%...以此類推）。
* **大小保底系統**：抽中五星時，有 50% 機率為當期限定（小保底）。若未抽中（歪常駐），下一次五星 100% 為當期限定（大保底）。
* **五星星芒補償（精確版）**：
  * **當期限定角色**：本輪模擬抽出的**第 1 隻（0命解鎖）不給予星芒**；第 2 隻開始（重複命座）每隻返還 2 抽資源（40 星芒）。
  * **常駐五星角色（歪掉）**：預設老玩家常駐池皆已解鎖，歪掉一律直接返還 2 抽資源（40 星芒）。

### 二、 五星光錐池（限定武器躍遷）
* **基礎機率**：單抽 0.8%。
* **硬保底**：80 抽必中五星。
* **軟保底（隱藏機制）**：從第 66 抽開始，每抽中獎機率增加 7%（第 66 抽 7.8%、第 67 抽 14.8%...以此類推）。
* **大小保底系統**：抽中五星時，有 75% 機率為當期限定光錐。若歪掉，下次五星 100% 為當期限定。
* **五星星芒補償**：不論是限定還是常駐光錐，只要抽中五星光錐，一律固定返還 2 抽資源（40 星芒）。

### 三、 四星物品與星芒回收機制（精確排他版）
四星的判定與回收是影響長期抽卡資源的關鍵，本模擬器實裝了以下精確邏輯：
* **基礎與保底**：四星單抽基礎機率為 5.1%，最多 10 抽必出四星。
* **五星優先權（順延機制）**：抽卡判定具備排他性。若該抽命中五星，將**直接跳過四星判定**。即使四星保底計數器已達 10，也會無損順延至下一抽，絕對不會發生「一抽內同時消耗五星與四星保底」的邏輯錯誤。
* **四星星芒轉化（碎片累加系統）**：
  * **非當期 UP 四星（50% 機率）**：固定獲得 0.4 抽（8 星芒）的碎片。
  * **當期 UP 四星（50% 機率）**：
    * 提供自訂「四星命座狀態」功能。
    * 若玩家設定為「UP 四星全滿命」，每次抽中 UP 四星直接獲得 1 完整抽（20 星芒）。
    * 若為「未滿命」，抽中時會隨機分配給 3 隻 UP 四星。若該角色在模擬過程中累計達到 6 命，後續抽中將轉為返還 1 抽；未滿 6 命前則返還 0.4 抽。
  * **碎片轉換**：所有 0.4 抽的碎片會在後台自動累加，一旦湊滿 1.0，即刻轉換為 1 次實際抽卡機會並投入當前循環。

### 四、 抽卡策略與資源消耗
* **資源轉換**：輸入的「星瓊」會以無條件捨去法（`Math.floor`）自動除以 160 並轉換為專票，與現有專票相加成為初始總抽數。
* **狀態繼承**：完美支援輸入當前的「已墊抽數」與「下次是否為大保底」狀態，並從該狀態接續精準計算。
* **策略優先級**：模擬器會將所有資源優先投入「角色池」，直到滿足目標角色數量後，才會將剩餘資源轉入「光錐池」。若雙目標皆達成，或資源徹底耗盡，即結算該次模擬。

### 五、 大數據機率呈現
* 採用蒙地卡羅方法，在瀏覽器端使用 JavaScript 引擎進行高達 100,000 次（可自訂）的獨立模擬計算。
* 最終報表捨棄「累積涵蓋率」的模糊概念，直接列出資源耗盡時，玩家**「確切停留在哪一個配置（如 3+0、4+1）」**的絕對獨立機率分布。
* 為維持版面整潔，發生機率低於 0.01% 的極端數據將被隱藏。
